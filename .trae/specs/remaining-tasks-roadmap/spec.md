# 剩余任务开发 Spec

## Why

项目已完成 Phase 1 的 Toast 通知系统和基础设计系统优化。现在需要继续完成剩余的所有核心功能和优化任务，排除社区功能（Task 9）和导师匹配功能（Task 10）作为未来迭代。本规范为剩余任务提供极其详细的技术要求和实现标准。

## What Changes

- 完成移动端响应式优化（Task 4）
- 完成性能优化（Task 5）
- 完成 AI 职业性格测评功能（Task 6）
- 完成智能实习推荐功能（Task 7）
- 完成成长路径规划功能（Task 8）
- 排除：Task 9 社区功能和 Task 10 导师匹配（保留未来实现）

## Impact

- **影响规格**: 响应式布局、性能优化、三个新功能模块
- **影响代码**: 
  - `src/components/` - 新增评估、推荐、路径可视化组件
  - `src/app/` - 新增页面路由
  - `src/lib/` - 新增业务逻辑库
  - `src/types/` - 新增类型定义
  - `src/hooks/` - 新增自定义 Hook

## ADDED Requirements

### Requirement: 移动端适配优化

系统 shall 提供完整的移动端体验：
1. 首页移动端单列布局，触控友好的卡片尺寸
2. 聊天界面移动端全屏体验，输入框固定在底部
3. 移动端侧边栏导航（抽屉式，从左侧滑入）
4. 所有表单移动端友好（大触控区域、虚拟键盘优化）
5. 移动端字体大小适配（最小 14px）

### Requirement: 性能优化

系统 shall 优化加载性能：
1. 实现图片懒加载 Hook（useIntersectionObserver）
2. 动态导入大型页面组件（next/dynamic）
3. API 响应缓存策略（SWR 或 React Query）
4. 首屏加载资源优化（字体、样式预加载）

### Requirement: AI 职业性格测评功能

系统 shall 提供职业性格测评：
1. 基于 Holland 职业兴趣理论（RIASEC 六维度）
2. 20-30 道测评题目（Likert 5 级量表）
3. 实时进度保存（防止意外刷新）
4. 测评结果可视化（雷达图 + 维度详情）
5. AI 解读和建议（基于测评结果生成个性化建议）
6. 结果保存并可在"我的信息"中查看

### Requirement: 智能实习推荐功能

系统 shall 提供实习推荐：
1. 基于用户画像（专业、城市、兴趣、测评结果）
2. 推荐算法匹配度计算（0-100 分）
3. 实习数据结构：职位、公司、地点、要求、匹配理由
4. 推荐结果列表展示（卡片式，可筛选）
5. 推荐详情页面（完整描述、匹配分析、申请建议）

### Requirement: 成长路径规划功能

系统 shall 提供成长路径可视化：
1. 路径数据结构：阶段、目标、行动项、时间线
2. 可视化组件：时间轴/甘特图样式
3. 基于用户当前状态生成个性化路径
4. 路径可编辑（添加/删除/调整行动项）
5. 进度跟踪（已完成/进行中/未开始）
